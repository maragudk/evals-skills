Available Skills

Step 1

trace-annotation-tool

Generate a custom local web app for open coding of LLM traces — the first qualitative pass of error analysis. Review traces one by one, write freeform observations about what went wrong, and mark each as pass, fail, or defer.

  • • Tailored trace rendering for your specific data format
  • • Keyboard shortcuts for fast annotation sessions
  • • Auto-saves to JSONL — close and resume anytime
  • • Built with FastHTML, TailwindCSS, and HTMX
Step 2

failure-taxonomy

Transform raw freeform annotations into a structured taxonomy of binary failure modes via axial coding. Groups similar observations into 3–7 coherent, non-overlapping, application-specific categories.

  • • Clusters open codes into actionable failure categories
  • • Re-labels every trace against the taxonomy
  • • Quantifies error rates and prioritizes by frequency
  • • Follows grounded theory methodology
Step 3

llm-as-a-judge

Build reliable automated evaluators that use an LLM to judge the outputs of another LLM pipeline. Each judge targets a single binary failure mode with a Pass/Fail verdict.

  • • Structured judge prompt design with few-shot examples
  • • Train/Dev/Test data splitting for prompt refinement
  • • TPR/TNR alignment metrics with iterative tuning
  • • Bias-corrected success rate estimation (Rogan-Gladen)
Cross-cutting

prompt-engineering

Craft, review, and improve prompts for LLM pipelines. Covers the two gulfs framework: specification (unclear instructions) vs. generalization (model limitations), and a seven-component prompt structure.

  • • Diagnose whether failures are specification or generalization gaps
  • • Seven-component prompt structure (role, instructions, context, ...)
  • • Iterative refinement process for task and judge prompts
  • • Anti-patterns and quick wins for common issues
